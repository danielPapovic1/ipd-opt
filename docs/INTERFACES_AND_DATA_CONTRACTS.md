# Interfaces and Data Contracts

This document formalizes the structures and function-level I/O contracts used across the IPD repository.

Related docs:
- [ARCHITECTURE.md](./ARCHITECTURE.md)
- [EXPERIMENTS.md](./EXPERIMENTS.md)

## 1. Canonical Structures

## 1.1 `Strategy` (`ipd_core.py:34-52`)

Fields:
- `name: str`
- `play_func: Callable`
- `is_bitstring: bool = False`
- `bitstring: Optional[str] = None`
- `memory_depth: int = 1`

Behavior:
- `play(history) -> Move` delegates to `play_func`.
- Equality and hash are name-based (`ipd_core.py:46-51`).

Contract implications:
1. Two strategies with the same `name` compare equal, independent of bitstring.
2. For bitstring strategies generated by `create_strategy_from_bitstring`, names are prefixed with first 10 bits (`ipd_core.py:251`), which is descriptive but not a guaranteed globally unique identifier.

## 1.2 `OptimizationResult` (`optimization.py:29-37`)

Fields:
- `best_strategy: Strategy`
- `best_fitness: float`
- `fitness_history: List[float]`
- `generation_history: List[int]`
- `time_taken: float`
- `parameters: Dict`
- `method: str`

Contract:
- All optimizers return this object from `.evolve(...)`.
- Downstream code expects these keys to exist for plotting/comparisons.

## 1.3 `MLResult` (`ml_prediction.py:29-38`)

Fields:
- `model_name: str`
- `accuracy: float`
- `precision: float`
- `recall: float`
- `f1: float`
- `feature_importance: Dict[str, float]`
- `train_size: int`
- `test_size: int`

Contract:
- `StrategyPredictor.train(...)` returns `List[MLResult]` (`ml_prediction.py:150-228`).

## 1.4 Tournament result dictionary (`ipd_core.py:95-121`)

Shape:
- `Dict[str, Dict]`
- Per strategy name:
  - `total_score: int|float`
  - `matches: int`
  - `scores: list`
  - `avg_score: float` (added post-loop)

Contract:
- `ExperimentRunner.evolved_vs_reference` converts this dictionary to `tournament_results.csv` (`experiments.py:442-455`).

## 2. Core Function Contracts

## 2.1 Game core (`ipd_core.py`)

### `IPDGame.play_match(strategy1, strategy2, num_rounds=100)` (`ipd_core.py:64-85`)
Input:
- Two `Strategy` objects.
- Positive integer rounds.

Output:
- `(score1, score2, history)` where history is `List[Tuple[Move, Move]]` from player1 perspective.

Rules:
- Strategy2 is evaluated on flipped history perspective (`ipd_core.py:76`).

### `IPDGame.round_robin_tournament(strategies, num_rounds=100, include_self_play=True)` (`ipd_core.py:88-121`)
Input:
- List of strategies.

Output:
- Tournament dictionary keyed by strategy names.

Important:
- Includes self-play by default (`ipd_core.py:90`, `ipd_core.py:100`).

### `create_strategy_from_bitstring(bitstring, memory_depth=1)` (`ipd_core.py:210-257`)
Input constraints:
- `len(bitstring) == 5` when depth=1.
- `len(bitstring) == 1 + 4^memory_depth` otherwise (`ipd_core.py:220-223`).

Output:
- `Strategy` with deterministic policy derived from bitstring.

Failure behavior:
- Assertion error on invalid bitstring length.

## 2.2 Evaluation and optimization (`optimization.py`)

### `FitnessEvaluator.evaluate(strategy, opponents=None, coevolution=False)` (`optimization.py:106-149`)
Output:
- Scalar fitness float.

Semantics:
- Pairwise average by default.
- Tournament average if `use_tournament_fitness=True` and not coevolution (`optimization.py:128-133`).
- Optional variance penalty subtracts `variance_penalty * std(scores)` (`optimization.py:137-140`).

### `FitnessEvaluator.evaluate_population(population, coevolution=False, coevolution_k=5)` (`optimization.py:151-170`)
Output:
- List of fitness values aligned to input population order.

Semantics:
- Optional threaded evaluation for non-coevolution and population size >= 10 (`optimization.py:157-160`).

### Optimizer `.evolve(...)` contracts
- GA: `GeneticAlgorithm.evolve` (`optimization.py:266-360`)
- EDA: `EDA.evolve` (`optimization.py:418-495`)
- HC: `HillClimbing.evolve` (`optimization.py:542-620`)
- TS: `TabuSearch.evolve` (`optimization.py:667-756`)

Common output:
- `OptimizationResult` with fully populated fields.

## 2.3 ML contracts (`ml_prediction.py`)

### `extract_features(strategy)` (`ml_prediction.py:41-91`)
Output keys (current implementation):
- `initial_cooperate`
- `response_to_cc`
- `response_to_cd`
- `response_to_dc`
- `response_to_dd`
- `coop_rate`
- `nice`
- `provokable`
- `forgiving`
- `tft_like`

### `generate_training_data(...)` (`ml_prediction.py:94-138`)
Inputs:
- `n_samples`, opponent list, `num_rounds`, `memory_depth`, `good_threshold_percentile`.

Outputs tuple:
- `X: np.ndarray`
- `y: np.ndarray`
- `feature_names: List[str]`
- `strategies: List[Strategy]`
- `fitnesses: List[float]`

Label contract:
- `y=1` iff fitness >= percentile threshold (`ml_prediction.py:135-136`).

### `StrategyPredictor.train(...)` (`ml_prediction.py:150-228`)
Inputs:
- `X`, `y`, `feature_names`, optional `test_size`, optional `memory_depth`.

Output:
- `List[MLResult]`.

Post-condition:
- `self.is_trained=True`, models and scaler populated.

### `StrategyPredictor.predict(strategy, model_name='RandomForest')` (`ml_prediction.py:231-245`)
Output:
- Positive-class probability float in `[0, 1]`.

Precondition:
- Model must be trained or raises `ValueError` (`ml_prediction.py:234-235`).

## 2.4 Experiment orchestration contracts (`experiments.py`)

### `run_parameter_tuning_ga(...)` (`experiments.py:101-147`)
Output:
- DataFrame with columns: `method,population_size,mutation_rate,best_fitness,time_taken,best_strategy`.

### `run_memory_depth_experiment(...)` (`experiments.py:150-233`)
Output:
- DataFrame with columns: `method,memory_depth,best_fitness,time_taken,strategy_bits`.

### `compare_all_methods(...)` (`experiments.py:235-353`)
Output:
- DataFrame with columns: `run,method,best_fitness,time_taken,final_gen`.

Side output:
- Convergence histories saved in `self.results_cache['method_histories']` (`experiments.py:352`).

### `evolved_vs_reference(...)` (`experiments.py:355-455`)
Output:
- DataFrame with columns: `strategy,avg_score,total_score,matches,type` sorted by descending `avg_score`.

Side output:
- `self.results_cache['evolved_best_by_method']` (`experiments.py:413-415`).

### `run_ml_experiments(...)` (`experiments.py:457-499`)
Output:
- DataFrame with columns: `train_size,model,accuracy,precision,recall,f1,memory_depth`.

## 4. Invariants and Assumptions

1. Strategy bitstring length is strictly tied to memory depth (`ipd_core.py:220-223`).
2. Strategy identity equality is name-based (`ipd_core.py:46-51`).
3. Tournament scoring uses same payoff matrix constants everywhere (`ipd_core.py:26-31`).
4. ML labels depend on percentile threshold (default 80th percentile in current calls).
5. Method comparison assumes exactly one row per method per run.
6. Current full suite writes artifacts to `results/` and does not version subfolders by timestamp.

## 5. Backward and Forward Compatibility Guidance

### Safe extensions
- New optimizers that return `OptimizationResult` and keep existing columns in result DataFrames.
- New ML models appended to existing `model` values in `ml_results.csv`.
- Additional plots/JSON files added without renaming existing outputs.

### Breaking changes to avoid
1. Renaming existing CSV columns consumed by report generation (`main.py:282-360`).
2. Changing output filenames used by post-processing (`main.py:187-225`, `experiments.py:503-701`).
3. Altering strategy equality semantics without revisiting tournament/result aggregation assumptions.

### Versioning recommendation
- If introducing schema changes, create `*_v2.csv`/`*_v2.json` and keep legacy artifacts for comparability in grading and paper citations.
